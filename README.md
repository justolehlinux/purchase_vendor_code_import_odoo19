# Purchase Import by Vendor Product Code — Odoo 19

## Функция 1: импорт заказа по коду поставщика

Добавляет поле строки заказа:

`Код товара поставщика для импорта` (`vendor_product_code_import`)

При импорте использует связку:

`Продавец заказа + Код товара продавца -> Товар Odoo`

Код должен заранее существовать во вкладке **Покупка** карточки товара.

CSV:

```csv
name,partner_id,order_line/vendor_product_code_import,order_line/product_qty,order_line/price_unit
PTEST-VENDOR-001,CHINASS,ABC-12345,24,0.8781
```

## Функция 2: автоматическая безопасная очистка дубликатов

1. Откройте **Покупка → Конфигурация → Прайс-листы поставщиков**.
2. Отфильтруйте поставщика, например CHINASS.
3. Выделите строки, которые нужно проверить.
4. Нажмите **Действие → Найти и удалить точные дубликаты**.
5. В окне предпросмотра проверьте результат и нажмите **Удалить точные дубликаты**.

Модуль удаляет только строки, в которых полностью совпадают:
поставщик, товар/вариант, код и название товара поставщика, количество, цена,
валюта, компания, даты действия, срок поставки, скидка, единица измерения и приоритет.

Если один код поставщика привязан к разным товарам, модуль покажет конфликт,
но не станет удалять его автоматически.


## Автоматические тесты Odoo

В папке `tests/` добавлены тесты Odoo `TransactionCase`. Они проверяют:

- подстановку товара при создании строки заказа по коду поставщика;
- импорт через стандартный ORM-механизм `load()` с колонкой
  `order_line/vendor_product_code_import`;
- запрет выбора товара по коду другого поставщика;
- остановку при конфликте: один код поставщика ведёт к разным товарам;
- удаление только точных дубликатов прайс-листа;
- сохранение ценовых ступеней и строк с изменённой ценой;
- отображение, но не автоматическое удаление конфликтов кода.

Тесты запускаются штатным тест-раннером Odoo, а не отдельной командой `pytest`,
поскольку они проверяют ORM, модели и импорт Odoo в транзакционной базе.
